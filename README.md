# My Task Tracker

Обновлённая версия трекера задач.

## Что изменено в этой версии

- На вкладке «День» навигация ограничена тремя днями: вчера, сегодня, завтра.
- Если нужна другая дата, её можно открыть через вкладку «Неделя», нажав на нужный день.
- На вкладке «Неделя» кнопки переключения периода подписаны: «Предыдущая» и «Следующая».
- В недельном календаре убран прогресс `2/2`, полоска и точка.
- Вместо прогресса под датой показываются иконки категорий и количество задач с указанным временем.
- Если в день нет задач с указанным временем, место под датой остаётся пустым.

## Локальный запуск

```bash
npm install
npm run dev
```

## Проверка сборки

```bash
npm run build
npm run typecheck
```

## Как обновить проект на GitHub вручную

Если репозиторий уже создан и файлы уже были загружены вручную через сайт GitHub:

1. Распакуйте этот архив на компьютере.
2. Откройте свой репозиторий на GitHub.
3. Нажмите **Add file** → **Upload files**.
4. Перетащите файлы и папки из распакованного проекта.
5. Если GitHub спросит про замену файлов, подтвердите обновление.
6. Внизу страницы в поле commit можно написать:

```text
Update day and week navigation
```

7. Нажмите **Commit changes**.

Не загружайте папки `node_modules` и `.next`, если они есть на вашем компьютере. Они не нужны для GitHub.

## Публикация через Vercel

После обновления файлов на GitHub Vercel обычно сам запустит новый деплой. Если деплой не начался автоматически, откройте проект в Vercel и нажмите **Redeploy**.
